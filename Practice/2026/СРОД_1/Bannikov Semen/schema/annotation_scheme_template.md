# ЛР2. Аннотационная схема (тема: виды птиц)

Документ фиксирует итоговое проектирование аннотационной схемы для ЛР2.

## 1. Область и состав схемы

### 1.1 Объекты предметной области

Размечаемые объекты: 10 видов птиц из `data/metadata/species_catalog.csv` (`SP001`-`SP010`).

### 1.2 Модальности

- Текст (`data/raw/text/`)
- Изображения (`data/raw/images/`)
- Аудио (`data/raw/audio/`)

### 1.3 Группы объектов x модальности


| Модальность | Идентификация вида | Морфология/внешний вид | Среда/контекст | Поведение   | Голос/звук | Метаданные источника |
| ----------- | ------------------ | ---------------------- | -------------- | ----------- | ---------- | -------------------- |
| Текст       | да                 | да                     | да             | да          | да         | да                   |
| Изображение | да                 | да                     | да             | ограниченно | нет        | да                   |
| Аудио       | да                 | косвенно               | да             | косвенно    | да         | да                   |


## 2. Требования к аннотациям по модальностям

### 2.1 Текст

Типы сущностей:

- `BIRD_SPECIES` - упоминание вида птицы.
- `HABITAT` - лес, парк, водоем и т.д.
- `BEHAVIOR` - полет, кормление, миграция и т.д.
- `VOCALIZATION` - пение, трель, крик и т.д.
- `COLOR_FEATURE` - признаки окраски.

Атрибуты:

- `species_id` (из `SP001`-`SP010`, если применимо)
- `canonical_form` (нормализованная форма)
- `source_doc_id`

### 2.2 Изображения

Типы аннотаций:

- `BirdInstance` (основной объект: птица конкретного вида).
- `BodyPart` (опционально: клюв, хвост, крыло).
- `VisualFeature` (опционально: цвет груди, полосы, пятна).

Атрибуты:

- `species_id`
- `shape_type` (`bbox` или `polygon`)
- `occluded` (да/нет)
- `truncated` (да/нет)
- `quality_flag` (`ok`, `blurred`, `far_object`)

### 2.3 Аудио

Типы аннотаций:

- `BirdCallSegment` (временной интервал целевого вида).
- `BackgroundSound` (дождь, ветер, другие птицы, антропогенный шум).

Атрибуты:

- `species_id`
- `start_time`, `end_time`
- `signal_type` (`song`, `call`, `unknown`)
- `overlap_flag` (перекрытие с другими источниками)

## 3. Формальные требования

- Единая сущность вида во всех модальностях: ключ `species_id`.
- Имена файлов:
  - текст: `SPxxx_text_yy.txt`
  - изображения: `SPxxx_img_yy.ext`
  - аудио: `SPxxx_aud_yy.ext`
- Форматы хранения аннотаций:
  - текст: CoNLL-U (допустимо TSV на этапе пилота);
  - изображения: CVAT XML/JSON (или Label Studio JSON);
  - аудио: TextGrid.
- В каждой аннотации обязательно поле источника (`source_url`/`source_id`) через манифест.

## 4. Содержательные требования

- Полнота по видам: каждый `species_id` покрыт 3 модальностями.
- Непротиворечивость: один и тот же фрагмент не может иметь взаимоисключающие метки.
- Минимальная детализация:
  - текст: минимум 1 фрагмент описания на вид;
  - изображения: минимум 3 изображения на вид;
  - аудио: минимум 3 записи на вид.
- Ограничения на теги:
  - не использовать омонимичные теги для разных сущностей;
  - использовать только согласованный список меток из этого документа.

## 5. Уровни аннотаций


| Модальность | Уровень 1           | Уровень 2                     | Уровень 3                                              |
| ----------- | ------------------- | ----------------------------- | ------------------------------------------------------ |
| Текст       | Документ            | Параграф                      | Токен + сущности (`BIRD_SPECIES`, `HABITAT`, ...)      |
| Изображение | Изображение целиком | Объект птицы (`BirdInstance`) | Детали (`BodyPart`, `VisualFeature`)                   |
| Аудио       | Запись              | Фрагмент                      | Сегмент события (`BirdCallSegment`, `BackgroundSound`) |
